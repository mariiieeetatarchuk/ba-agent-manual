# User Story Template for EliteA

## Required Fields

- Name
- Description
- Business Value
- Acceptance Criteria
- BDD Scenarios
- UI/UX Notes
- Dependencies
- Assumptions
- Out of Scope
- Links to Documentation
- Figma References

## Description Format

AS a user, I WANT to [perform an action], SO THAT [business value or user benefit].

## Acceptance Criteria Format

Acceptance criteria must be clear, testable, and written as a numbered list.

## BDD Scenario Format

Given [initial context]
When [action]
Then [expected result]

## UI/UX Notes

Use Figma design data to identify buttons, labels, fields, states, tooltips, navigation elements, and user-facing messages.

## Dependencies

Mention related documentation, integrations, technical dependencies, or linked pages.

## Out of Scope

Clearly define what is not included in the current story.
