# User Story Template for EliteA

## Required Fields

* Name
* Description
* Business Value
* Acceptance Criteria
* BDD Scenarios
* UI/UX Notes
* Dependencies
* Assumptions
* Out of Scope
* Links to Documentation
* UI References

## Description Format

AS a user, I WANT to [perform an action], SO THAT [business value or user benefit].

## Acceptance Criteria Format

Acceptance criteria must be clear, testable, and written as a numbered list.

Example:

1. The user can perform the main action successfully.
2. The system validates required fields.
3. The system shows a clear success or error message.
4. The created or updated item is available for further work.

## BDD Scenario Format

BDD scenarios must follow the Given / When / Then format.

Example:

Given the user is on the required screen, when the user performs the action, then the expected result is displayed.

## UI/UX Notes Format

UI/UX notes should describe visible UI elements, required fields, validation behavior, buttons, dropdowns, and user feedback messages.

## Dependencies Format

Dependencies should include required system components, APIs, permissions, documentation, or related features.

## Assumptions Format

Assumptions should describe what is considered true for the user story to work correctly.

## Out of Scope Format

Out of scope should describe what is not included in the current user story.

## Links to Documentation Format

Documentation links should reference related Confluence pages, GitHub manuals, or other project documentation used for the story.

## UI References Format

UI references should describe predefined UI details used by the agent, such as screens, buttons, input fields, dropdowns, validation messages, and page elements.

Note: This POC does not use a real Figma integration. UI context is represented through predefined mock UI details inside the pipeline configuration.
